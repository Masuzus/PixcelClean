# 智能背景移除与像素颜色规整设计

## 1. 产品目标

Pixel Clean 用于把统一背景的 AI 图片清理为可复用的透明像素图素材。当前版本自动恢复被背景污染的边缘颜色和覆盖率，不再提供需要用户选择辉光色、强度和阈值的独立辉光工具。

核心约束：

1. 完全透明像素统一为 `#00000000`；
2. 自然边缘允许 Alpha `1..254`，硬像素边缘只输出 `0` 或 `255`；
3. 处理和导出直接操作 8 位 RGBA，不经过缩放采样；
4. 启用颜色规整时，实体 RGB 颜色数量不得超过用户设置的上限，Alpha 覆盖率不作为新 RGB 色；
5. 颜色面板按实体 RGB 合并展示颜色和像素次数，不为 Alpha 生成独立色卡；
6. 预览、统计和导出必须使用同一份处理结果。

## 2. 非目标

当前版本不支持：

- 需要语义识别的烟雾、玻璃或毛发抠图；
- 手工指定辉光颜色和混合模式；
- 加色或发光混合模式；
- 抖动生成的伪色阶；
- 自动把复杂摄影图片转换成高质量手绘像素图。

## 3. 处理管线

```text
原始 PNG
→ 背景色及背景波动范围估计
→ 保护蒙版应用
→ OKLab 双阈值背景区域识别
→ 高可信前景代表色提取
→ 线性 RGB 边缘反算与 RGB 去污染
→ 自然 Alpha 或硬边 Alpha 输出
→ 相似颜色聚类
→ 每个像素替换为代表色
→ 完整颜色统计
→ 预览与透明 PNG 导出
```

优先级：

1. 保护蒙版阻止像素被背景移除；
2. 已有半透明输入在自然模式中原样保留；
3. 高可信背景像素变为透明，可信混色边缘恢复前景 RGB 和覆盖率；
4. 所有非透明像素参与 RGB 颜色规整；
5. 预览背景只影响右侧画布显示，不写入 PNG。

## 4. 智能背景移除

### 4.1 背景模型

背景色默认取四角样本的中位数，也允许用户手工覆盖。处理阶段把角落样本转换到 OKLab，排除明显前景后估计背景波动范围，并生成严格阈值和宽松阈值。严格匹配像素作为背景种子，宽松匹配像素只有与种子连通时才扩展为背景。

开启“移除封闭背景”后，图像内部的严格背景候选也可作为种子。保护蒙版始终优先，禁止被背景区域生长覆盖。

### 4.2 自动前景代表色

从明显远离背景且未被删除的像素中提取最多 48 个全局高可信前景候选。反算每个边缘像素时，再从六像素邻域向主体内部寻找最多 32 个距离背景更远的局部候选。局部候选优先解决多色图案和渐变中全局色卡覆盖不足的问题，全程不要求用户选择前景色或辉光色。

### 4.3 边缘反算

背景边界内外三像素进入边缘反算。设输入颜色为 `C`、背景为 `B`、前景候选为 `F`，在线性 RGB 中求解：

```text
C ≈ αF + (1 - α)B
```

只有重建误差足够小、且 `F` 比 `C` 更远离背景时才接受结果。接受后输出 RGB 为未受背景污染的 `F`：自然边缘输出连续 Alpha，硬像素边缘按 50% 覆盖率二值化。不可信结果保持原像素，避免无依据删除主体细节。

## 5. 相似颜色替换

### 5.1 输入与输出

颜色规整只处理 Alpha 大于 `0` 的像素。透明像素不计入调色板颜色上限。

用户设置最大实体颜色数，范围为 `8..20`，默认 `16`。如果原图颜色数没有超过上限，不修改 RGB。

### 5.2 感知色差

不能直接使用 sRGB 欧氏距离，因为数值距离不等于人眼感知距离。每个唯一 RGB 转换到 OKLab，再进行聚类和最近颜色匹配。

OKLab 距离只能用于同一色系内部。颜色先硬性分为中性色、红、暖色、绿、青、蓝和紫七类；每类至少保留一个代表色，任何像素都禁止映射到其它色系。该约束用于防止红色像素被替换成视觉距离接近的紫色。

### 5.3 聚类流程

1. 精确统计全部非透明 RGB 及出现次数；
2. 使用确定性的远点初始化选择最多 N 个中心；
3. 在 OKLab 中执行带像素次数权重的迭代聚类；
4. 每个聚类选择最接近中心的原图实际颜色作为代表色；
5. 将每个非透明像素替换为最近的代表色；
6. 重新统计并确认实体颜色数不超过上限。

代表色必须来自原图已有颜色，避免聚类中心取整后产生新的偶然颜色。默认不使用抖动，因为抖动会破坏像素图的干净色块并增加局部噪点。

### 5.4 统计信息

每次处理返回：

- 背景移除像素数；
- 规整前实体颜色数；
- 规整后实体颜色数；
- 被替换的像素数；
- 完成 RGB 去污染的边缘像素数；
- 半透明像素数。

## 6. 全部像素颜色

颜色面板从最新 `processedImage` 实时派生，不单独保存到工程文件。

- 面板按唯一 `#RRGGBB` 展示，不截取前 N 项；
- Alpha 大于 `0` 的同 RGB 像素合并计数；
- Alpha 等于 `0` 的完全透明像素不展示；
- 每项显示该 RGB 在所有非透明覆盖率下的像素总次数；
- 所有色卡次数之和必须等于非透明像素总数；
- 工程重新打开后自动处理并重新生成统计。

Alpha 仍按像素精确保存在处理结果和导出 PNG 中，只是不作为色卡维度展示。

颜色列表支持两种排序：按使用次数，以及按 OKLab 明度从深到浅。深到浅排序中透明色位于最前，实体色按明度升序、色相和十六进制值稳定排序。

## 7. 自动处理

- 参数或蒙版变化后立即把结果标记为过期；
- 停止修改 `500 ms` 后自动执行一次完整处理；
- 新修改到来时取消过期 Worker 和待执行任务；
- 用户取消或算法报错后不自动循环重试；
- 保留“强制高质量渲染”按钮；
- 导出按钮只在结果为最新状态时启用。

## 8. 像素精确预览

- `100%` 表示一个原始像素对应一个 CSS 像素；
- `- / +` 按钮使用整数倍率 `1x..16x`；
- 滚轮允许连续缩放，非整数倍率仅用于浏览；
- Canvas 保持原始像素缓冲区，禁止使用 `transform: scale()` 放大；
- 整数倍率下使用 Point/Nearest-Neighbor 显示；
- 平移位置对齐设备像素。

右侧预览背景支持棋盘格、黑色、白色和自定义颜色。切换背景不得改变处理数据、触发重新处理或影响 PNG 透明度。

## 9. 工程文件

当前工程版本为 `3`：

```json
{
  "kind": "pixel-clean-project",
  "version": 3,
  "source": {
    "fileName": "sprite.png",
    "pngDataUrl": "data:image/png;base64,..."
  },
  "editor": {
    "tolerance": 20,
    "includeEnclosedAreas": true,
    "imageView": { "scale": 4, "offsetX": 12, "offsetY": -18 },
    "brushSize": 12,
    "interactionMode": "paint",
    "protectedMask": "base64...",
    "autoPreviewEnabled": true,
    "previewBackground": {
      "mode": "checkerboard",
      "customColor": "#6B7280"
    },
    "backgroundColor": "#1C1523",
    "edgeMode": "natural",
    "paletteSortMode": "dark-to-light",
    "paletteReduction": {
      "enabled": true,
      "maximumColors": 16
    }
  }
}
```

迁移规则：

- v1 工程补充默认背景色和 16 色规整；
- v2 工程读取原图、背景参数和保护蒙版，忽略旧辉光蒙版与辉光参数；
- v3 工程恢复边缘模式和颜色规整设置；缺少 `edgeMode` 时使用 `natural`；
- 任何校验失败时保留当前编辑状态。

处理结果和颜色统计不保存到工程文件，因为它们可以由原图、蒙版和参数确定性重建。

## 10. 快捷键与历史

- `Command/Ctrl + S`：保存当前工程；
- `Command/Ctrl + Z`：撤销；
- `Command/Ctrl + Shift + Z` 或 `Ctrl + Y`：重做；
- 连续画笔和滑块变化合并为一个历史事务；
- 历史记录包含背景参数、边缘模式、保护蒙版、颜色规整设置和预览背景；
- 撤销后发生新编辑时清空重做栈；
- 首版最多保留 20 条历史。

## 11. 验收标准

- 128 个近似实体颜色设置上限 16 后，导出实体颜色数不超过 16；
- 禁用颜色规整时，只有可信边缘反算可以修改 RGB；
- 自然边缘能够恢复中间 Alpha，硬像素边缘只能输出 `0` 或 `255`；
- 已知背景上的合成边缘能够恢复接近原始的前景 RGB 和 Alpha；
- 全透明像素必须为 `#00000000`；
- 颜色统计次数总和等于导出 PNG 中 Alpha 大于 `0` 的像素总数；
- 颜色统计中的 RGB 与导出 PNG 解码后的非透明像素一致；
- 相同输入和参数重复处理得到完全相同的像素数据；
- v1、v2、v3 工程均可安全打开。
